# Project Documentation Index

This directory is the central hub for all project documentation. All files MUST be classified according to the following structure:

### Directories

| Directory | Content Type | Purpose |
| :--- | :--- | :--- |
| `adr/` | Decision Records | Documents architectural decisions and the *why* behind them. |
| `assets/` | Static Media | Diagrams, images, or files referenced by other documentation. |
| `learning/` | Tutorials | Project-based learning material. Currently a Go course taught through the backend codebase. Start at `learning/README.md`. |
| `research/` | Exploratory | Technical deep-dives, spikes, market analysis, protocol specs. Use `research/TEMPLATE.md`. |
| `rfc/` | Proposals | Future roadmaps, project plans, feature proposals requiring review. |

---
*If you are adding a new document, determine its purpose based on the table above and place it in the corresponding directory.*
